# Order Accounting Module

Невеликий модуль обліку замовлень для бізнес-системи.

## Стек

* Python
* Flask
* SQLite
* SQLAlchemy
* HTML/Jinja2

## Що реалізовано

* створення клієнта;
* створення товару;
* створення замовлення;
* автоматичний розрахунок суми замовлення;
* перегляд замовлень конкретного клієнта;
* перевірка основних бізнес-правил.

## Бізнес-правила

* замовлення не можна створити без клієнта;
* у замовленні має бути товар;
* кількість товару має бути більшою за 0;
* не можна замовити більше товару, ніж є на складі;
* сума замовлення рахується автоматично на основі ціни товару та кількості.

## Структура проєкту

```text
app.py              — запуск Flask-додатку
extensions.py       — підключення SQLAlchemy
models.py           — моделі бази даних
routes.py           — маршрути сторінок
services.py         — бізнес-логіка
templates/          — HTML-шаблони
requirements.txt    — залежності
```

## Основні таблиці

* `Client` — клієнти.
* `Product` — товари.
* `Order` — загальна інформація про замовлення.
* `OrderItem` — товари всередині замовлення.

`OrderItem` використовується для того, щоб одне замовлення могло містити один або кілька товарів. Також у ньому зберігається ціна товару на момент створення замовлення.

## Запуск проєкту

1. Перейти в папку проєкту:

```bash
cd TestTask
```

2. Створити віртуальне середовище:

```bash
python -m venv venv
```

3. Активувати його:

```bash
venv\Scripts\activate
```

4. Встановити залежності:

```bash
pip install -r requirements.txt
```

5. Запустити проєкт:

```bash
python app.py
```

Після запуску сайт буде доступний за адресою:

```text
http://127.0.0.1:5000/
```

## Чому обраний такий підхід

Я розділив код на моделі, маршрути та бізнес-логіку, щоб проєкт був зрозумілішим і його було легше підтримувати.
Окремо винесена логіка створення замовлення, бо саме там знаходяться основні бізнес-правила: перевірка клієнта, товару, кількості та автоматичний розрахунок суми.
